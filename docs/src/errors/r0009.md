# REGLE_R0009: createVariant unmatched discriminant

Code: `REGLE_R0009`
Level: warn

## What this error means

Regle reported **createVariant unmatched discriminant** while running validation. This diagnostic helps you find misconfigured rules, state, or integration setup before it silently breaks form behaviour.

## Why this happens

- The runtime detected an unexpected value or configuration related to this area (Runtime).
- See the terminal message for the exact path, rule name, or parameter values involved.

## How to fix it

Follow the `fix:` hint in the terminal output. The diagnostic catalog in `@regle/core` defines the recommended action for this code.

``ts
// Wrong — discriminant without literal()
{ type: { required }, email: { required } }

// Correct
{ type: { literal: literal('EMAIL') }, email: { required, email } }
``
